# Environment Variables Setup

This project uses environment variables to store sensitive Supabase credentials. They should never be hardcoded.

## Local Development

1. Create a `.env.local` file in the root directory:
```bash
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anonymous-key-here
```

2. Build the config file locally:
```bash
npm run build
```

This generates `config.js` from your environment variables.

## Vercel Deployment

1. **Push to GitHub** (without `.env.local` - it's in `.gitignore`)

2. **In Vercel Dashboard:**
   - Go to your project settings
   - Navigate to **Settings → Environment Variables**
   - Add two variables:
     - **Name:** `VITE_SUPABASE_URL`  
       **Value:** Your Supabase project URL (from Supabase dashboard)
     - **Name:** `VITE_SUPABASE_ANON_KEY**  
       **Value:** Your Supabase anonymous key (from Supabase dashboard Settings → API)

3. **Deploy:**
   - Vercel will automatically run `npm run build` during deployment
   - This generates `config.js` with your environment variables
   - Your credentials are never visible in the public repo

## Security Notes

- ✅ `VITE_SUPABASE_ANON_KEY` can be public (it's limited to anonymous access)
- ✅ Environment variables are injected at **build time**
- ✅ Credentials never appear in version control
- ✅ Different deployments can have different credentials

## File References

- `.env.example` - Template showing what variables are needed
- `.env.local` - Your local credentials (gitignored)
- `config.js` - Generated file loaded by your app (generated by build script)
- `build.js` - Script that generates config.js from environment variables
